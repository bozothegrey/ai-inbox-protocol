# AGENTS.md

This is the operating protocol for Stefano's AI Inbox workflow.

Any AI system, automation, script, or agent that creates or modifies notes in the Notion `AI Inbox` should read this file before writing.

## System model

The system separates raw material, AI processing, long-term knowledge, and operating rules.

| Layer | Location | Purpose |
|---|---|---|
| Raw/source archive | Google Drive | PDFs, screenshots, article exports, transcripts, source files, raw captures |
| AI working layer | Notion `AI Inbox` | AI notes, mobile captures, source summaries, working notes, review queue, promotion candidates |
| Curated knowledge base | Obsidian / Personal Wiki | Reviewed permanent notes |
| Protocol and templates | GitHub `ai-inbox-protocol` | Rules, templates, workflow docs, agent instructions |

## Core principle

Notion is the cockpit and working layer. It is not the final Personal Wiki.

Raw/source files stay in Google Drive by default.

Reviewed permanent notes eventually move to the Personal Wiki / Obsidian.

GitHub is the canonical source of workflow rules.

## Notion database

Main database:

```text
AI Inbox
```

Expected properties:

| Property | Type | Purpose |
|---|---|---|
| `Name` | Title | Note title |
| `Author` | Select | Who created or imported the note |
| `Type` | Select | Kind of note |
| `Status` | Status | Workflow stage |
| `Review` | Select | Review state |
| `Domain` | Multi-select | Topic or project domain |
| `Source` | URL | Original source URL, if any |
| `Raw Location` | URL | Google Drive raw/source link, if any |
| `Target` | Select | Intended destination or archive target |
| `Created` | Created time | Automatic creation timestamp |
| `Last Edited` | Last edited time | Automatic edit timestamp |

## Author policy

Set `Author` accurately.

Use:

- `Sergio` for notes generated or substantially edited by Sergio.
- `Stefano` for notes written directly by Stefano.
- `Other AI` for notes produced by another AI assistant or tool.
- `Imported` for material imported from another system without substantial editing.

When Sergio creates a page, set:

```text
Author = Sergio
Review = Pending
```

## Type policy

Use the most accurate `Type`.

Recommended values:

- `AI Note` — structured note generated by Sergio or another AI.
- `Mobile Capture` — quick mobile capture, OCR, link, screenshot summary, or rough idea.
- `Source Summary` — summary of an external source, document, article, email, or raw material.
- `Working Note` — synthesis, map, analysis, intermediate thinking, link suggestions.
- `Promotion Candidate` — cleaned material ready to move into the Personal Wiki / Obsidian after review.

## Status and review policy

Use `Status` for workflow stage and `Review` for human review state.

Current Notion status groups may use standard labels such as:

- `Not started`
- `In progress`
- `Done`

Use `Review` to track review state:

- `Pending`
- `Reviewed`
- `Rejected`

Do not mark material as ready for the Personal Wiki unless it is a reviewed `Promotion Candidate`.

## Domain policy

Use `Domain` to make the note easy to retrieve and summarize.

Recommended values:

- `BoGa`
- `Fitness`
- `Music`
- `Sovereign Debt`
- `Tech`
- `Reading`
- `Family`

Use multiple domains when appropriate, but do not turn tags into confetti. Classification is useful only while it remains boring.

## Source and raw-location policy

Raw/source material lives in Google Drive by default.

Examples:

- PDFs
- screenshots
- exported articles
- email exports
- transcripts
- source documents
- images
- raw notes before processing

When a source exists, set:

```text
Source = original URL or reference
Raw Location = Google Drive raw/source link, if known
```

Do not silently import raw files into Notion or Obsidian.

Notion should store processed notes, summaries, OCR text, comments, maps, drafts, indexes, and review notes.

## Standard page structure

New structured notes should normally use this body structure:

```markdown
## Summary

Short summary of the note.

## Main Note

Structured content.

## Links / Suggested Links

Suggested related notes, concepts, projects, or future links.

## Source / Raw Location

Source URL, Google Drive raw location, or note that no raw source exists.

## Review / Next Actions

- [ ] Keep
- [ ] Edit
- [ ] Promote
- [ ] Archive
```

For source summaries, prefer:

```markdown
## Source

## Why It Matters

## Summary

## Key Points

## Concepts / Entities

## Suggested Links

## Next Actions
```

For promotion candidates, prefer:

```markdown
## Summary

## Final Note

## Suggested Destination

## Source / Raw Location

## Review
```

## Promotion policy

The Personal Wiki / Obsidian is the curated destination.

Notion is not the canonical archive.

A note should become a `Promotion Candidate` before it is moved to the Personal Wiki.

After promotion, update the Notion page accordingly and avoid maintaining two competing canonical versions.

## Minimal fallback

If this protocol cannot be fully consulted, create only a minimal capture note and mark it for review.

Minimum fields:

```text
Name = clear title
Author = accurate author
Type = Mobile Capture or AI Note
Review = Pending
```

Minimum body:

```markdown
## Summary

## Captured Content

## Source / Raw Location

## Review / Next Actions
```
